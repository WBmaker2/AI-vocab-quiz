# Student Progression Design

Date: 2026-03-25
Status: Draft

## Goal

학생들이 한 번 하고 끝나는 것이 아니라, 자신의 성장과 수집 보상을 보며 더 자주 돌아오고 더 오래 활동하도록 만든다.

이번 설계에서는 세 활동 전체에 공통으로 붙는 두 가지 축을 먼저 도입한다.

1. 개인 최고 기록 시스템
2. 배지·스티커 수집 시스템

둘 다 학생 이름 기준으로 Firebase에 저장해 다음 활동 때도 이어지게 한다.

## Scope

이번 범위에 포함한다.

- 듣기 퀴즈 완료 화면의 개인 최고 기록 비교
- 말하기 연습 완료 화면의 개인 최고 기록 비교
- 짝 맞추기 완료 화면의 개인 최고 기록 비교
- 세 활동 공통 배지 획득 판정
- 학생 이름 기준 Firebase 프로필 저장
- 결과 화면에서 새 기록과 새 배지 표시

이번 범위에서 제외한다.

- 출석 보상
- 오늘의 도전 과제
- 반 전체 협동 목표
- 교사용 학생 배지 대시보드
- 학생 로그인 계정 체계

## Product Intent

리더보드는 다른 학생과의 비교를 위한 기능이다. 하지만 초등학생에게는 리더보드보다 “지난번의 나보다 나아졌는가”가 더 강한 반복 동기가 된다.

그래서 이번 설계는 보상 구조를 두 층으로 나눈다.

- 리더보드: 경쟁 보상
- 개인 최고 기록 + 배지: 성장 보상

이 두 층이 함께 있어야 상위권 학생뿐 아니라 대부분의 학생도 반복 플레이 동기를 얻는다.

## Recommended Approach

추천 접근은 `학생 프로필 레이어`를 Firebase에 추가하는 방식이다.

학생 이름과 학교, 학년을 묶은 하나의 프로필 문서에 활동별 최고 기록과 획득 배지를 저장한다. 각 활동은 완료 시 이 프로필을 읽고, 새 결과가 더 좋으면 갱신하며, 배지 조건을 만족하면 같은 문서에 반영한다.

이 접근을 선택한 이유:

- 세 활동이 하나의 게임처럼 연결된다.
- 리더보드와 별개로 모든 학생에게 개인 보상을 줄 수 있다.
- 이후 오늘의 도전 과제, 누적 활동 수, 연속 학습 같은 기능으로 확장하기 쉽다.

## Data Model

새 컬렉션을 추가한다.

- `studentProfiles/{profileId}`

추천 `profileId` 규칙:

- `schoolId__grade__studentNameNormalized`

문서 예시 필드:

- `schoolId`
- `schoolName`
- `grade`
- `studentName`
- `studentNameNormalized`
- `totalSessions`
- `listeningBestScore`
- `listeningBestCorrectCount`
- `speakingBestScore`
- `speakingBestCorrectCount`
- `matchingBestScore`
- `matchingBestTime`
- `earnedBadges`
- `createdAt`
- `updatedAt`

`earnedBadges`는 문자열 배열 또는 badge metadata map으로 시작할 수 있다. 초기 구현은 단순성을 위해 배열을 우선 추천한다.

예시:

- `earnedBadges: ["first_challenge", "listening_star", "matching_speed"]`

## Identity Rule

학생 프로필의 범위는 다음 조합으로 고정한다.

- 학교
- 학년
- 학생 이름

이 뜻은 다음과 같다.

- 같은 이름이어도 학년이 다르면 다른 학생 프로필이다.
- 같은 이름이어도 학교가 다르면 다른 학생 프로필이다.
- 학생 로그인 없이도 교실 운영이 가능하다.

한계:

- 같은 학교, 같은 학년에서 동명이인이 있으면 같은 프로필로 합쳐질 수 있다.

이번 단계에서는 이 제약을 수용한다. 추후 필요하면 교사용 별칭 코드나 학생 번호를 도입해 분리할 수 있다.

## Personal Best System

세 활동이 끝날 때 공통 규칙으로 개인 최고를 비교한다.

### Listening

비교 기준:

- 최고 점수
- 최고 정답 수

결과 화면 예시:

- `개인 최고 점수 갱신!`
- `지난번보다 2문제 더 맞혔어요`
- `이번에는 최고 기록과 같아요`

### Speaking

비교 기준:

- 최고 점수
- 최고 정답 수

결과 화면 예시:

- `말하기 최고 기록 갱신!`
- `지난번보다 더 정확하게 말했어요`

### Matching

비교 기준:

- 최고 점수
- 최고 시간

결과 화면 예시:

- `개인 최고 점수 갱신!`
- `지난번보다 12초 빨라졌어요`
- `다음엔 1초만 줄여도 새 기록이에요`

## Badge System

초기 버전에서는 규칙이 단순하고 설명이 쉬운 배지들만 도입한다.

추천 초기 배지:

- `first_challenge`
  - 아무 활동이든 첫 완료
- `listening_star`
  - 듣기 퀴즈에서 높은 기준 달성
- `speaking_bravery`
  - 말하기 연습 1회 완료
- `matching_speed`
  - 짝 맞추기에서 기준 시간 이내 완료
- `practice_keeper`
  - 누적 활동 수 기준 달성

초기 기준은 너무 촘촘하게 만들지 않는다. 학생이 배지 규칙을 직관적으로 이해할 수 있어야 한다.

이번 첫 릴리스에서 실제로 shipping 하는 배지는 아래 다섯 개로 제한한다.

- `first_challenge`
- `listening_star`
- `speaking_bravery`
- `matching_speed`
- `practice_keeper`

`unit_master`는 가치가 크지만, 활동 간 완료 상태를 더 안정적으로 추적해야 하므로 다음 단계로 미룬다.

초기 릴리스에서 쓰는 구체 기준 예시:

- `matching_speed`: 45초 이내 완료

## UI Flow

### Activity completion

각 활동 완료 시 공통 흐름:

1. 현재 점수 계산
2. 학생 이름 확인 또는 입력
3. 학생 이름 기준 프로필 조회
4. 최고 기록 비교
5. 새 기록이면 갱신
6. 배지 조건 판정
7. 새 배지 획득 시 저장
8. 완료 화면에 아래를 표시
   - 최종 점수
   - 개인 최고 비교 메시지
   - 새로 획득한 배지
   - 기존 리더보드 등록 영역

중요한 원칙:

- progression 저장은 리더보드 참여 여부와 분리한다.
- 학생이 리더보드 등록을 거절해도 개인 최고 기록과 배지는 저장될 수 있어야 한다.
- 리더보드는 경쟁 레이어, progression은 성장 레이어로 독립 동작해야 한다.

### Result screen additions

완료 화면에 추가할 블록:

- `내 성장 기록`
  - 최고 점수
  - 지난 기록 대비 변화
- `이번에 얻은 배지`
  - 새 배지 카드 또는 배지 칩

## Suggested Visual Direction

리더보드 영역보다 개인 최고/배지 영역의 감정 톤을 더 따뜻하게 둔다.

- 개인 최고: 성공 강조, 자신감 메시지
- 배지: 수집하고 싶은 작은 보상 느낌
- 리더보드: 경쟁 톤

즉, 결과 화면은 아래 순서가 좋다.

1. 최종 점수
2. 개인 성장 피드백
3. 새 배지
4. 리더보드 등록

이 순서가 학생에게 가장 긍정적인 마무리 경험을 준다.

## Firebase Rules Direction

읽기와 쓰기 규칙은 다음을 기준으로 확장한다.

- 학생 프로필 읽기: 학교/학년 범위의 공개 학생 활동 흐름에서 허용
- 학생 프로필 쓰기: 학생 활동 완료 화면에서 자기 이름 기준 점수/배지 저장 허용

주의:

현재 앱 구조는 학생 브라우저가 직접 Firebase에 쓰는 구조라 완전한 위변조 방지는 어렵다. 이번 기능도 같은 한계를 가진다.

추후 더 안전하게 만들려면:

- Firebase callable function
- 서버 검증 후 저장

같은 구조가 필요하다.

## Edge Cases

- 학생 이름이 비어 있으면 개인 기록 저장 불가
- Firebase 연결이 없으면 개인 기록과 배지 영역은 비활성 안내 표시
- 첫 활동이면 비교 메시지 대신 `첫 기록입니다` 표시
- 새 점수가 낮아도 배지 조건은 만족할 수 있음
- 동명이인 문제는 현재 단계에서 허용
- 세 활동은 가능한 한 같은 학생 이름 입력 상태를 공유해, 같은 학생이 활동마다 다른 프로필로 갈라지지 않게 한다.

## Rollout Strategy

구현은 한 번에 넓게 넣지 않고 아래 순서를 추천한다.

1. `studentProfiles` 데이터 구조 추가
2. 짝 맞추기에 먼저 개인 최고 + 배지 적용
3. 듣기/말하기로 공통 확장
4. `unit_master` 같은 단원 단위 배지는 다음 단계에서 추가
5. 필요하면 학생 시작 화면에 `내 배지` 요약 추가

이 순서를 택하는 이유는 짝 맞추기가 현재 점수 구조가 가장 선명하고, 결과 화면도 가장 풍부해서 첫 적용 대상으로 적합하기 때문이다.

## Success Criteria

이 기능이 잘 들어갔다고 판단하는 기준:

- 학생이 같은 이름으로 다시 플레이했을 때 이전 최고 기록과 비교 메시지가 나온다.
- 새 최고 기록이면 Firebase에 저장되고 다음 결과 화면에서 반영된다.
- 배지 조건 달성 시 즉시 획득 메시지가 나온다.
- 세 활동 모두 같은 학생 프로필 레이어를 공유한다.
- 기존 리더보드 흐름과 충돌하지 않는다.
