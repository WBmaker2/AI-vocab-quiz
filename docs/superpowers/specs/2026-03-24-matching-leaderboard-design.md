# Matching Leaderboard Design

## Summary

짝 맞추기 게임 완료 화면에 학교/학년별 리더보드를 추가한다. 리더보드는 `이 주의 영단어 왕`, `이 달의 영단어 왕`, `올해의 영단어 왕` 3개 기간으로 운영하며, 완료 화면에서 탭으로 전환해 확인한다.

학생은 짝 맞추기 게임을 완료한 뒤 `리더보드에 점수를 등록하시겠습니까?` 확인을 받고, 동의하면 이름을 입력해 점수를 등록할 수 있다. 같은 학교와 학년에서 같은 학생 이름으로 이미 저장된 기록이 있을 경우, 주간/월간/연간 각각 현재 점수가 더 높을 때만 해당 리더보드를 갱신한다.

## Goals

- 학교와 학년 기준으로 분리된 주간/월간/연간 리더보드를 제공한다.
- 짝 맞추기 게임 완료 직후 자연스럽게 점수 등록을 유도한다.
- 같은 학생의 더 낮은 점수는 중복 저장하지 않는다.
- 완료 화면에서 리더보드 탭을 전환하며 즉시 결과를 볼 수 있게 한다.

## Non-Goals

- 듣기 퀴즈와 말하기 연습 점수는 리더보드에 포함하지 않는다.
- 학생 계정 시스템이나 로그인 기반 식별은 도입하지 않는다.
- 이름 충돌을 해결하기 위한 별도 학번/코드 체계는 이번 범위에 포함하지 않는다.
- 학생 모드 기본 진입 화면에 상시 리더보드를 노출하지 않는다.

## User Experience

### 1. 게임 완료 후 점수 등록 흐름

1. 학생이 짝 맞추기 게임을 완료한다.
2. 완료 화면에 최종 점수, 문제 수, 걸린 시간이 보인다.
3. `리더보드에 점수를 등록하시겠습니까?` 확인 UI를 표시한다.
4. 학생이 동의하면 이름 입력 모달 또는 카드 영역이 열린다.
5. 이름을 입력하고 저장하면:
   - 주간, 월간, 연간 각 기간의 기존 최고 점수와 비교한다.
   - 현재 점수가 더 높을 때만 해당 기간 리더보드를 갱신한다.
   - 더 높지 않으면 그 기간은 변경하지 않는다.
6. 저장 후 완료 화면 아래 리더보드 영역이 최신 상태로 갱신된다.

### 2. 리더보드 표시 방식

- 리더보드는 `짝 맞추기 게임 완료 화면`에만 표시한다.
- 리더보드 영역은 탭형 UI로 구성한다.
  - `이 주의 영단어 왕`
  - `이 달의 영단어 왕`
  - `올해의 영단어 왕`
- 기본 탭은 `이 주의 영단어 왕`으로 한다.
- 각 탭에는 상위 순위를 보여준다.
- 데이터가 없으면 `아직 등록된 기록이 없습니다.`를 표시한다.

### 3. 리더보드 항목 정보

각 항목에는 다음 정보를 표시한다.

- 순위
- 학생 이름
- 점수
- 걸린 시간(보조 정보)

점수는 숫자 뒤에 `점`을 붙여 현재 앱 톤과 맞춘다.

## Scope and Identity Rules

### Scope

리더보드는 아래 범위로 분리된다.

- 학교: `schoolId`
- 학년: `grade`
- 기간 종류: `week` / `month` / `year`
- 기간 키: 해당 주, 월, 연도를 식별하는 값

즉, 같은 학교라도 학년이 다르면 리더보드가 다르다.

### Student identity

- 학생 식별은 `studentName` 문자열로 처리한다.
- 동일 학교, 동일 학년, 동일 기간, 동일 학생 이름이면 같은 엔트리로 취급한다.
- 학생 이름 비교를 위한 정규화 값 `studentNameNormalized`를 함께 저장한다.

## Data Model

Firebase Firestore에 리더보드 전용 경로를 추가한다.

### Collection path

`matchingLeaderboards/{scopeKey}/entries/{studentNameNormalized}`

### Document ID

문서 id는 아래 정보를 조합한 고정 문자열로 생성한다.

상위 document id는 `scopeKey`, 하위 entry document id는 `studentNameNormalized`를 사용한다.

이 구조를 쓰면 같은 학생의 같은 기간 기록을 덮어쓸 수 있고, 조회도 현재 scope 하위 컬렉션만 읽으면 된다.

### Document shape

```json
{
  "scopeKey": "encoded-school-id__3__week__2026-W13",
  "schoolId": "encoded-school-id",
  "schoolName": "도촌초등학교",
  "grade": "3",
  "studentName": "김민준",
  "studentNameNormalized": "김민준",
  "periodType": "week",
  "periodKey": "2026-W13",
  "score": 2349,
  "elapsedSeconds": 51,
  "solvedPairs": 24,
  "updatedAt": "serverTimestamp",
  "createdAt": "serverTimestamp"
}
```

## Period Calculation

기간 키는 한국 시간대 기준으로 계산한다.

- week: `YYYY-Www`
- month: `YYYY-MM`
- year: `YYYY`

리더보드 데이터는 클라이언트에서 현재 날짜를 기준으로 현재 기간 키를 계산해 사용한다.

## Update Rules

점수 저장 시 아래 규칙을 적용한다.

1. 현재 학교와 학년이 없는 경우 저장하지 않는다.
2. 학생 이름이 비어 있으면 저장하지 않는다.
3. 주간/월간/연간 각각 별도로 처리한다.
4. 해당 문서가 없으면 새로 생성한다.
5. 해당 문서가 있으면 기존 `score`와 비교한다.
6. 현재 점수가 기존보다 높을 때만 `score`, `elapsedSeconds`, `solvedPairs`, `updatedAt`를 갱신한다.
7. 현재 점수가 같거나 낮으면 그대로 둔다.

즉, 한 번의 저장 시도에서:

- 주간만 갱신될 수 있고,
- 월간과 연간은 유지될 수 있으며,
- 세 기간 모두 갱신될 수도 있다.

## UI Structure

### WordMatchingGame completion section

완료 화면에 아래 요소를 추가한다.

- `리더보드에 점수를 등록하시겠습니까?` 질문 영역
- `등록하기` / `건너뛰기` 또는 동등한 확인 버튼
- 이름 입력 UI
- 저장 결과 메시지
- 리더보드 탭 섹션

### Leaderboard tabs

- 탭 3개를 가로로 배치한다.
- 현재 탭을 강조 표시한다.
- 탭 아래에는 리스트를 표시한다.
- 모바일에서는 세로 스택 또는 가로 스크롤 가능한 탭 형태를 허용한다.

## Loading and Refresh Behavior

- 완료 화면 진입 시 현재 학교와 학년 기준 리더보드를 불러온다.
- 점수 저장 성공 후 3개 기간 리더보드를 다시 불러온다.
- 저장 실패 시 기존 리더보드는 유지하고 오류 메시지를 보여준다.

## Error Handling

- Firebase 미설정 시: 저장 버튼 비활성화 또는 안내 메시지 표시
- 학교/학년 정보 없음: 리더보드 섹션 대신 등록 불가 안내 표시
- 학생 이름 없음: 저장 실행 전 입력 요구
- 저장 실패: `점수 등록 중 오류가 발생했습니다.` 같은 사용자용 메시지 제공
- 조회 결과 없음: 빈 상태 안내 표시

## Security and Rules Expectations

리더보드는 학생이 로그인 없이 등록하므로 쓰기 정책을 신중히 제한해야 한다.

이번 범위에서 필요한 최소 조건:

- `matchingLeaderboards`는 필요한 필드만 쓰기 허용
- 허용 범위는 현재 앱이 저장하는 문서 구조로 제한
- 앱은 `scopeKey`를 이용해 학교, 학년, 기간 기준으로 조회
- Firestore rules는 공개 읽기 자체를 막지 않고 문서 스키마를 제한한다
- Firestore rules는 `scopeKey`와 document id가 학교, 학년, 기간, 학생 이름 정규화 값과 일치하는지도 확인한다
- Firestore query filter를 규칙에서 완전히 강제하기 어렵기 때문에, 범위 제한은 앱의 `scopeKey` 조회를 기준으로 한다

정확한 Firestore 규칙은 구현 시 현재 프로젝트 규칙과 함께 조정한다.

현재 구조는 학생이 브라우저에서 직접 Firestore에 점수를 기록하는 방식이므로, 완전한 위변조 방지는 불가능하다. 진짜 무결성이 필요하면 서버 또는 callable function을 통한 서명된 저장 흐름이 추가로 필요하다.

## Verification Plan

구현 후 아래를 검증한다.

1. 짝 맞추기 완료 화면에 리더보드 질문과 탭 영역이 표시된다.
2. 학생 이름 입력 후 저장 시 주간/월간/연간 문서가 생성된다.
3. 같은 학생의 더 낮은 점수는 기존 기록을 덮어쓰지 않는다.
4. 같은 학생의 더 높은 점수는 해당 기간 기록만 갱신한다.
5. 탭 전환 시 각 기간의 순위 목록이 바뀐다.
6. 학교 또는 학년이 바뀌면 다른 범위의 리더보드가 표시된다.
7. `npm run build`가 통과한다.

## Open Decisions Resolved

- 리더보드는 학생 모드 기본 화면이 아니라 `게임 완료 후`에만 표시한다.
- 3개 리더보드는 `탭형 UI`로 제공한다.
- 동일 학생 판별은 로그인 기반이 아니라 `학생 이름 문자열` 기준으로 한다.
- 저장 대상은 `짝 맞추기 게임 점수`만 포함한다.
