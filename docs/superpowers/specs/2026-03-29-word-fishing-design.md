# 단어 낚시 게임 설계 문서

작성일: 2026-03-29

## 목표

기존 `듣기 퀴즈`, `말하기 퀴즈`, `짝 맞추기`, `학급 빙고`와 다른 감각의 반응형 학생 활동으로 `단어 낚시` 게임을 추가한다.

핵심 목표는 다음과 같다.

- 화면을 떠다니는 여러 카드 중에서 정답 뜻만 빠르게 낚아채는 손맛을 만든다.
- 기존 단어 세트(`vocabularySets`)를 그대로 재활용한다.
- 학생이 개인 플레이로 바로 시작할 수 있도록 `TTS 읽기 모드`만 지원한다.
- 짝 맞추기보다 더 동적이고, 시선 추적과 반응 속도가 함께 들어가는 게임 감각을 만든다.
- 현재 학생 모드 흐름에 자연스럽게 붙어야 한다.

## 기본 가정

이번 설계는 다음을 기본 가정으로 둔다.

- 첫 버전은 `학생 모드 개인 활동`으로 시작한다.
- 학생은 현재 불러온 단어 세트로 게임에 들어간다.
- 단어 낚시는 `영어 단어를 듣고`, 화면을 떠다니는 `뜻 카드(한국어)` 중 정답만 누르는 방식이다.
- 첫 버전은 개인 플레이 기준이므로 `TTS 읽기 모드`만 지원한다.
- 같은 기기에서 학생이 게임을 하는 상황이 기본이지만, 교실에서 프로젝터/공용 화면으로도 운영할 수 있게 UI를 구성한다.

## 추천 접근

추천 구조는 `학생 모드 반응형 아케이드 게임 + 공통 단어 세트 재사용`이다.

이 구조를 선택한 이유:

- 현재 앱 구조와 가장 잘 맞는다. `학생 모드 -> 단어 세트 불러오기 -> 개별 게임 시작` 흐름을 그대로 유지할 수 있다.
- `짝 맞추기`와 달리 움직이는 목표를 누르는 게임 감각을 줄 수 있다.
- Firebase 실시간 세션 없이도 첫 버전을 빠르게 구현할 수 있다.
- 나중에 필요하면 `교실 전체 경쟁형`으로 확장할 수 있는 기반을 남길 수 있다.

## 고려한 대안

### 1. 학생 모드 개인 반응형 게임

학생 각자 현재 불러온 단어 세트로 게임을 진행한다. 영어 단어를 듣고, 화면을 떠다니는 뜻 카드 중 정답만 누른다.

- 장점: 현재 앱에 가장 쉽게 붙는다.
- 장점: 기존 단어 세트/학생 활동 구조를 거의 그대로 쓴다.
- 장점: 구현 범위가 작고 안정적이다.
- 단점: 교실 전체 실시간 동기화는 없다.

### 2. 교사 호스트형 실시간 낚시 세션

교사가 단어를 선택하거나 읽고, 여러 학생 기기에서 동시에 반응한다.

- 장점: 교실 전체 경쟁감이 강하다.
- 장점: 선생님이 직접 읽는 흐름과 가장 잘 맞는다.
- 단점: `빙고`와 비슷한 실시간 세션 구조를 또 하나 만들게 되어 첫 버전 범위가 커진다.

### 3. 단일 기기 순번형 교실 게임

프로젝터나 대형 화면에서 한 명씩 나와 정답을 누르는 방식이다.

- 장점: 구현은 단순하다.
- 단점: 동시에 여러 학생이 참여하는 감각이 약하고, 앱의 개인 활동 흐름과도 멀어진다.

이번 문서에서는 `1. 학생 모드 개인 반응형 게임`을 MVP로 채택한다.

## MVP 범위

첫 버전은 아래 범위로 제한한다.

- 게임 종류: `단어 낚시`
- 진입 위치: 학생 모드 게임 버튼 영역, `단어 짝 맞추기`와 `학급 빙고 게임` 사이
- 사용 데이터: 현재 불러온 단어 세트 1개
- 카드 표시: `뜻 카드(한국어)`가 떠다님
- 문제 제시:
  - `TTS 읽기 모드`
- 반응 방식: 학생이 움직이는 뜻 카드를 눌러 정답을 낚음
- 점수 요소:
  - 정답
  - 오답
  - 반응 속도
- 결과 표시:
  - 최종 점수
  - 정답 수
  - 반응 시간 요약

이번 MVP에서 제외:

- Firebase 실시간 멀티플레이
- 교사 호스트 전용 낚시 세션
- 리더보드 연동
- 학생 성장 기록 연동
- 장애물, 폭탄, 스킬 아이템
- 난이도 다단계 시스템

## 핵심 사용자 흐름

### 1. 학생 진입 흐름

1. 학생은 학생 모드에서 학교/선생님/학년/단원을 고른다.
2. `단어 세트 불러오기`로 현재 단어 세트를 준비한다.
3. `단어 낚시` 버튼을 누른다.
4. 게임 시작 전 간단한 안내를 본다.
5. `게임 시작`을 누르면 TTS 읽기 모드로 바로 시작한다.

### 2. 한 라운드 진행 흐름

1. 시스템이 현재 단어 세트에서 목표 영어 단어 하나를 고른다.
2. 화면 위에는 여러 개의 뜻 카드가 물고기처럼 떠다닌다.
3. 목표 단어는 TTS로 자동 재생된다.
4. 학생은 화면을 움직이는 뜻 카드 중 정답만 눌러 잡는다.
5. 정답이면 점수와 짧은 성공 연출을 준다.
6. 오답이면 감점 또는 시간 페널티를 준다.
7. 제한 시간 안에 반응하지 못하면 놓친 것으로 처리한다.
8. 다음 목표 단어로 넘어간다.

### 3. 종료 흐름

1. 정해진 라운드 수를 마치거나 단어 풀을 모두 사용하면 게임이 끝난다.
2. 결과 화면에서 아래를 보여준다.
   - 최종 점수
   - 맞춘 문제 수
   - 평균 반응 속도 또는 전체 소요 시간
   - 다시 하기
   - 홈으로

## 게임 메커니즘

## 1. 카드 구성

한 라운드에는 `1개의 정답 카드 + 여러 개의 오답 카드`가 함께 떠다닌다.

권장 시작 규칙:

- 화면 동시 카드 수: `6장`
- 구성:
  - 정답 1장
  - 오답 5장

오답 카드는 같은 단어 세트에서 무작위로 뽑는다.

이유:

- 카드가 너무 적으면 낚시 게임 느낌이 약하다.
- 너무 많으면 초등학생 기준으로 시선 부담이 커진다.
- 6장은 모바일과 태블릿, PC 모두에서 균형이 좋다.

## 2. 카드 움직임

카드는 정적인 grid가 아니라 `부드럽게 떠다니는 fish lane` 느낌으로 움직인다.

권장 방식:

- 좌우 이동 + 약한 상하 흔들림
- 카드별로 속도와 지연 시간을 조금씩 다르게 줌
- 화면 밖으로 완전히 사라지기보다는 보이는 영역 안에서 계속 순환

첫 버전에서는 복잡한 physics 대신 `CSS keyframes + 위치 랜덤화`가 적절하다.

## 3. 입력 규칙

- 학생은 언제든 카드 하나를 누를 수 있다.
- 정답이면 즉시 다음 라운드로 넘어간다.
- 오답이면 카드가 짧게 흔들리거나 색이 바뀐다.
- 한 라운드에서 정답이 처리되면 다른 카드는 잠깐 멈춘 뒤 다음 라운드로 교체한다.

## 4. 시간 규칙

권장 시작 규칙:

- 라운드 제한 시간: `10초`
- 제한 시간 초과 시 `놓침` 처리
- 전체 라운드 수: `10문제`

이유:

- 너무 짧으면 저학년이 어렵다.
- 10초는 움직이는 카드와 TTS를 함께 따라가기에 조금 더 여유가 있다.
- 10문제는 수업 중 짧게 한 판 돌리기에 적당하다.

## 5. 점수 규칙

권장 시작 규칙:

- 정답: `+100점`
- 빠른 정답 보너스:
  - 2초 이내 `+40점`
  - 4초 이내 `+20점`
- 오답: `-30점`
- 시간 초과: `0점`

핵심은 `속도 + 정확도` 둘 다 보상하는 것이다.

## 문제 제시 모드

이번 MVP는 `TTS 읽기 모드`만 사용한다.

- 라운드 시작 시 앱이 목표 영어 단어를 자동으로 읽는다.
- `다시 듣기` 버튼으로 반복 재생 가능
- 학생 혼자서도 바로 시작할 수 있어야 하므로 별도 모드 선택 단계는 두지 않는다.

## 화면 설계

### 학생 화면

학생 화면은 다음 4개 영역으로 구성한다.

1. `상단 게임 정보`
   - 남은 문제 수
   - 현재 점수
   - 남은 시간

2. `현재 목표 단어 영역`
   - 목표 영어 단어 표시
   - `다시 듣기`

3. `떠다니는 카드 영역`
   - 6개의 뜻 카드가 움직이는 메인 플레이 영역

4. `결과/피드백 영역`
   - 정답
   - 오답
   - 놓침

### 교사 사용성

MVP는 별도 교사 호스트 화면을 만들지 않는다. 학생 개인 플레이에 바로 들어갈 수 있게 `TTS 전용` 흐름으로 단순화한다.

## 아키텍처 결정

## 추천 구성

- 새로운 학생 게임 컴포넌트 `WordFishingGame`
- 순수 유틸 `wordFishing.js`
- 기존 `useSpeechSynthesis` 재사용
- Firebase 실시간 저장 없음
- 현재 로드된 학생 단어 세트(`library.student.items`)만 사용

이 구성이 좋은 이유:

- 현재 앱 구조와 가장 잘 맞는다.
- 복잡한 원격 상태 없이 반응형 게임 감각을 빠르게 구현할 수 있다.
- 이후 리더보드나 성장 기록을 붙일 때도 확장성이 있다.

## 오류 처리

- 현재 단어 세트가 2개 미만이면 게임 시작 불가
- TTS 미지원 브라우저에서는 게임 시작을 막고 안내 문구 표시
- 움직임 애니메이션이 너무 과한 기기에서는 `reduced motion` 대응
- 클릭 가능한 카드가 너무 빨라서 누르기 어렵지 않도록 속도 상한 설정

## 테스트 기준

- 학생 화면에서 `단어 낚시` 버튼으로 진입 가능
- TTS로 단어가 읽히고 카드가 정상적으로 움직임
- 정답 선택 시 점수와 다음 라운드가 정상 반영
- 오답과 시간 초과가 정상 처리
- 10문제 완료 후 결과 화면 표시
- 모바일 세로 화면, 태블릿 가로 화면, PC 화면 모두 레이아웃이 무너지지 않음

## 단계적 확장 후보

첫 버전 이후에는 아래 확장이 자연스럽다.

- 개인 최고 점수 저장
- 단어 낚시 리더보드
- 폭탄 카드/보너스 카드
- 교사 호스트형 실시간 낚시 세션
- 학생 랜덤 발표형 낚시 라운드
