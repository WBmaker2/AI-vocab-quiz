# Teacher Publisher And Copy Design

## Goal

교사 관리 화면에 학년별 출판사 선택 기능을 추가하고, 공개된 다른 학교 단어카드를 현재 학교 단어카드로 안전하게 복사할 수 있게 만든다. 기존 단어 등록, 엑셀 일괄 등록, 공개 여부, 병합 규칙은 유지한다.

## Current Context

- 교사 화면은 현재 `학교와 선생님 정보` -> `학년과 단원 선택` -> `엑셀로 단원 일괄 등록` -> 단어 입력/목록 순서다.
- 단어세트 저장 단위는 `ownerUid + grade + unit`이고 Firestore `vocabularySets` 컬렉션에 저장된다.
- 교사 프로필은 `teachers/{uid}`에 저장된다.
- 엑셀 업로드는 현재 선택 학년의 여러 단원을 한꺼번에 병합 저장한다.
- 중복 병합 기준은 `word + meaning`이다.

## User Requirements

1. 교사 화면에서 `학년과 단원 선택`과 `엑셀로 단원 일괄 등록` 위치를 서로 바꾼다.
2. 교사는 학년별로 영어교과서 `출판사`를 선택할 수 있어야 한다.
3. 출판사는 자유 입력이 아니라 고정 목록에서 선택해야 한다.
4. `엑셀로 단원 일괄 등록` 아래에 `다른 학교 단어카드 복사` 기능을 추가한다.
5. 복사 기능은 `현재 선택한 학년`만 기준으로 검색한다.
6. 검색 결과는 `출판사` 기준으로 찾고, 선택한 결과를 현재 학교 카드로 복사할 수 있어야 한다.
7. 추가 엑셀 등록과 동일하게 중복은 안전하게 병합되어야 한다.
8. `엑셀로 단원 일괄 등록` 카드에서도 같은 학년별 출판사를 바로 선택할 수 있어야 한다.
9. `다른 학교 단어카드 복사` 검색 결과에는 기능 점검을 위해 우리 학교 카드도 함께 보여야 한다.

## Fixed Publisher Options

학년별 출판사 선택 목록은 아래 고정 옵션을 사용한다.

- 동아출판 윤여범
- 미래엔 강정진
- 아이스크림미디어 박유미
- 와이비엠 김혜리
- 와이비엠 최희경
- 천재교과서 함순애
- 천재교과서 김태은
- 천재교육 이동환
- 동아출판 정은숙
- 비상교육 우길주

## Recommended Approach

출판사를 `학년별 교사 메타데이터`와 `단원별 vocabularySets 문서`에 함께 저장한다.

이 접근을 추천하는 이유:

- 교사 UI에서는 `3학년 출판사`처럼 자연스럽게 다룰 수 있다.
- 복사 검색은 Firestore에서 `grade + publisher + published` 기준으로 직접 필터링할 수 있어 단순하다.
- 기존 `grade/unit` 단위 저장 구조를 유지하면서도 기능을 확장할 수 있다.

대신 출판사 문자열이 각 단원 문서에 중복 저장되지만, 이 정도 중복은 검색 단순성과 구현 안정성에 비해 비용이 작다.

## Data Model

### `teachers/{uid}`

기존 필드에 아래 필드를 추가한다.

- `gradePublishers: map<string, string>`

예시:

```json
{
  "teacherName": "김영어",
  "schoolId": "seoul-elementary",
  "schoolName": "서울초등학교",
  "isActive": true,
  "gradePublishers": {
    "3": "와이비엠 최희경",
    "4": "천재교육 이동환"
  }
}
```

### `vocabularySets/{setId}`

기존 필드에 아래 필드를 추가한다.

- `publisher: string`

예시:

```json
{
  "ownerUid": "teacher-uid",
  "schoolId": "school-id",
  "schoolName": "도촌초등학교",
  "teacherName": "김홍년",
  "grade": "4",
  "unit": "2",
  "publisher": "와이비엠 최희경",
  "published": true,
  "sourceType": "xlsx",
  "items": []
}
```

## Section Order

교사 메인 관리 화면의 순서를 아래처럼 바꾼다.

1. 학교와 선생님 정보

7. 단어 목록

## `엑셀로 단원 일괄 등록`

기존 동작은 유지한다.

- 상단에 `출판사` 드롭다운을 추가한다.
- 이 출판사 드롭다운은 아래 `학년과 단원 선택` 섹션의 출판사와 같은 학년별 값을 공유한다.
- 즉, 위에서 바꾸면 아래도 즉시 같은 값으로 보이고, 아래에서 바꾸면 위도 같은 값으로 보인다.
- 현재 선택 학년 기준으로 여러 Lesson 단원을 병합 저장
- 중복은 `word + meaning` 기준으로 건너뜀
- 현재 `학생 공개` 체크 상태를 반영
- 현재 학년의 공통 출판사도 모든 반영 단원에 함께 저장한다.

## `다른 학교 단어카드 복사`

엑셀 등록 카드 바로 아래에 새 섹션을 추가한다.

표시 요소:

- 현재 선택 학년 안내
- 출판사 선택 드롭다운
- `검색` 버튼
- 검색 결과 목록
- 선택 결과 복사 버튼

검색 규칙:

- 현재 선택 학년만 대상으로 검색한다.
- 출판사는 위 고정 목록에서 선택한다.
- 검색 대상은 `published == true` 인 공개 세트만 허용한다.
- 기능 점검과 비교를 위해 `우리 학교` 결과도 함께 보여준다.
- 다만 결과 카드에서 `우리 학교` 여부를 시각적으로 구분해 보여준다.

검색 결과는 `학교 + 선생님 + 출판사 + 학년` 단위로 묶는다.

각 결과 카드에 표시할 정보:

- 학교 이름
- 선생님 이름
- 출판사
- 학년
- 포함 단원 목록
- 공개 단어 총 개수
- `우리 학교` 또는 `다른 학교` 표시 배지

복사 규칙:

- 사용자가 검색 결과 하나를 선택하고 `우리 학교 카드로 복사`를 누르면 실행된다.
- 선택 결과의 `현재 학년 공개 단원들`을 모두 읽어 현재 교사 계정의 같은 학년 세트로 복사한다.
- 같은 단원이 이미 있어도 덮어쓰지 않고 병합한다.
- 병합 기준은 기존과 동일하게 `word + meaning`
- 중복은 건너뛰고 새 단어만 추가한다.
- 현재 교사 화면의 `학생 공개` 체크 상태를 복사된 모든 단원에 공통 적용한다.
- 현재 학년 출판사가 비어 있으면 선택한 출판사로 자동 채운다.
- 현재 학년 출판사가 이미 있으면 유지한다.

복사 완료 메시지에는 아래 정보를 포함한다.

- 반영 단원 수
- 추가된 새 단어 수
- 건너뛴 중복 단어 수

## `학년과 단원 선택`

이 섹션은 엑셀/복사 섹션 아래로 이동한다.

추가 요소:

- `출판사` 드롭다운

동작:

- 학년을 바꾸면 해당 학년의 저장된 출판사를 불러온다.
- 출판사를 바꾸면 현재 학년의 draft 상태에 반영되고, 위 `엑셀로 단원 일괄 등록` 카드의 출판사 드롭다운과 즉시 동기화된다.
- 현재 단원 저장 시 선택된 출판사가 단원 문서에 함께 저장된다.
- 단어세트 불러오기 시, 해당 학년의 저장된 출판사와 현재 단원의 출판사를 함께 표시/동기화한다.

## Publisher-Centered UX

카드를 등록하는 사람과 복사하는 사람 모두가 출판사 정보를 쉽게 다룰 수 있어야 한다.

등록자 UX:

- 학년을 선택하면 마지막으로 사용한 출판사를 자동으로 보여준다.
- 출판사 선택 UI는 엑셀 등록 카드와 단원 선택 카드 양쪽에 보여서, 등록 직전에 바로 확인하고 바꿀 수 있다.
- 출판사가 비어 있으면 `엑셀 가져오기`, `현재 단원 저장`, `복사 실행`을 막고 같은 문구로 안내한다.

복사자 UX:

- 현재 학년의 출판사가 이미 잡혀 있으면 `다른 학교 단어카드 복사`의 검색 출판사도 자동으로 같은 값으로 채운다.
- 검색 결과에 우리 학교 카드도 같이 보여서, 같은 출판사 데이터가 실제로 존재하는지 바로 확인할 수 있다.
- 우리 학교 카드는 배지로 구분해 테스트용 확인과 비교가 쉽도록 한다.
- 장기적으로는 가장 많이 쓰는 출판사 순으로 상단 추천 표시를 붙일 수 있지만, 이번 범위에서는 고정 드롭다운 + 자동 채움까지만 포함한다.

## Data Flow

## Source Of Truth Rule

학년별 출판사의 공식 기준값은 `teachers.gradePublishers[grade]`이다.

규칙:

- 교사 화면에서 출판사 드롭다운을 바꿔도 즉시 Firestore에 저장하지 않는다. 변경값은 교사 화면의 local draft state로만 유지한다.
- `현재 단원 저장`, `엑셀 일괄 저장`, `다른 학교 카드 복사` 중 하나를 실행할 때만, 그 시점의 draft 출판사를 `teachers.gradePublishers[grade]`에 확정 저장한다.
- 이후 같은 액션에서 저장되는 모든 단원은 그 시점의 확정 출판사 값을 `publisher`로 함께 저장한다.
- 이미 저장되어 있던 기존 단원은 출판사 선택을 바꿨다고 즉시 일괄 수정하지 않는다.
- 다만 같은 학년 단원을 다시 저장하거나 복사/엑셀 병합으로 다시 쓰는 순간, 새 기준 출판사로 덮어써서 점진적으로 정렬한다.

즉, `gradePublishers[grade]`가 현재 기준이고, `vocabularySets.publisher`는 검색과 표시를 위한 저장본 캐시다.

### 교사 프로필 로드

1. `teachers/{uid}`에서 `gradePublishers`를 읽는다.
2. 현재 선택 학년의 출판사를 교사 화면 state에 반영한다.
3. 같은 출판사 state를 `엑셀 등록`, `단원 선택`, `출판사 복사 검색` UI가 함께 공유한다.
3. 현재 학년의 `gradePublishers[grade]` 값이 있으면 그 값이 항상 editor 초기값보다 우선한다.
4. `gradePublishers[grade]` 값이 비어 있고, 불러온 현재 단원 세트에 `publisher` 값이 있으면 그 값을 editor 초기값으로 보여준다.
4. 이 경우에는 화면 표시용 초기값만 채우고, `gradePublishers[grade]` 자체는 사용자가 저장하기 전까지 자동 저장하지 않는다.

### 현재 단원 저장

1. 현재 학년, 단원, 출판사, 공개 여부, items를 모은다.
2. `gradePublishers[grade]`를 teacher profile에 저장한다.
3. `vocabularySets/{ownerUid__grade__unit}`에 `publisher`를 포함해 저장한다.

### 엑셀 일괄 저장

1. 현재 선택 학년의 출판사를 확인한다.
2. 각 단원 저장 시 같은 `publisher`를 함께 저장한다.
3. 중복 병합은 기존 규칙을 유지한다.

### 다른 학교 카드 복사

1. 현재 선택 학년과 선택 출판사로 검색한다.
2. 검색 결과는 `우리 학교`와 `다른 학교`를 모두 포함한 공개 세트를 반환한다.
3. 선택한 결과의 단원별 items를 모두 읽는다.
4. 현재 교사 계정의 같은 학년 단원들에 병합 저장한다.
5. 필요하면 현재 학년 출판사도 함께 저장한다.

## Firestore Query Design

필요한 새 조회는 아래 두 가지다.

1. `searchPublishedPublisherSources(grade, publisher)`
   - 현재 학년 + 출판사 + 공개된 세트 기준 조회
   - 현재 학교도 포함
   - 결과는 `schoolId + ownerUid` 기준으로 묶어서 반환

2. `copyPublishedPublisherSourceToTeacher(...)`
   - 선택된 source의 단원 목록을 읽고
   - 현재 교사 세트에 병합 저장

## Legacy Data Compatibility

기존 데이터에는 `gradePublishers` 또는 `publisher`가 없을 수 있다.

호환 규칙:

- 기존 `teachers` 문서에 `gradePublishers`가 없으면 빈 객체로 간주한다.
- 기존 `vocabularySets` 문서에 `publisher`가 없으면 학생/교사 일반 불러오기 기능은 그대로 허용한다.
- 새 `출판사 기반 복사 검색`은 `publisher`가 있는 공개 세트만 대상으로 한다.
- 따라서 예전 공개 세트를 복사 검색 대상으로 쓰려면, 해당 교사가 그 학년 단원을 한 번 다시 저장하거나 엑셀/복사 기능으로 재저장해 `publisher`를 채워야 한다.
- 구현에서는 기존 단어세트를 불러왔을 때 현재 학년 출판사가 비어 있고, 세트에도 `publisher`가 없으면 출판사 미설정 상태로 표시한다.
- 구현에서는 기존 단어세트를 불러왔을 때 `gradePublishers[grade]`는 비어 있지만 현재 단원 세트의 `publisher`만 존재하면, 그 `publisher`를 임시 editor 값으로 먼저 보여준다.
- `gradePublishers[grade]`와 현재 단원 세트의 `publisher`가 둘 다 존재하고 서로 다르면, `gradePublishers[grade]`를 editor 표시값과 다음 저장의 기준값으로 사용한다.
- 이 임시 editor 값은 자동 마이그레이션을 일으키지 않는다. 사용자가 `현재 단원 저장`, `엑셀 등록`, `다른 학교 카드 복사` 중 하나를 실행했을 때만 `gradePublishers[grade]`와 새로 저장되는 단원들에 확정 반영된다.
- 현재 학년 출판사를 선택하고 저장하는 순간부터 그 학년 데이터는 새 구조로 점진 전환된다.

## Firestore Rules Impact

Rules 업데이트 필요:

- `teachers` create/update 허용 필드에 `gradePublishers` 추가
- `vocabularySets` create/update 허용 필드에 `publisher` 추가
- 타입 검증:
  - `gradePublishers`는 map
  - `publisher`는 string

## Error Handling

- 현재 학년이 없으면 복사 검색을 막고 안내 메시지를 표시한다.
- 출판사를 선택하지 않으면 검색 버튼을 비활성화한다.
- 검색 결과가 없으면 `해당 출판사의 공개 카드가 없습니다.` 메시지를 표시한다.
- 복사 중 실패하면 기존 teacher items/state는 유지하고 오류 메시지만 표시한다.
- 출판사가 없는 상태에서 현재 단원 저장 또는 엑셀 복사를 시도하면 저장을 막고 `출판사를 먼저 선택하세요.`를 표시한다.
- 우리 학교 카드가 검색되더라도 일반 결과와 같은 방식으로 선택/복사할 수 있게 두되, 카드에 `우리 학교` 배지를 붙여 오인만 막는다.

## Acceptance Checks

아래 항목은 구현 완료 후 pass/fail로 확인해야 하는 공식 검증 기준이다.

- 교사 화면에서 `엑셀로 단원 일괄 등록`이 `학년과 단원 선택`보다 위에 보인다.
- 학년별로 고정 목록 중 하나의 출판사를 선택할 수 있다.
- 엑셀 등록 카드와 단원 선택 카드의 출판사 값이 항상 동기화된다.
- 학년을 바꾸면 해당 학년의 출판사가 따로 유지된다.
- 현재 단원 저장 시 단원 문서에 `publisher`가 저장된다.
- 엑셀 일괄 등록 시 반영 단원 모두에 같은 출판사가 저장된다.
- 다른 학교 카드 복사 섹션에서 현재 학년 기준으로만 출판사 검색이 된다.
- 검색 결과에는 우리 학교 카드도 함께 보인다.
- 우리 학교 결과는 배지로 구분된다.
- 검색 결과를 복사하면 현재 학교 같은 학년 단원으로 안전하게 병합된다.
- 기존 단어와 겹치는 항목은 중복 없이 유지된다.

### End-To-End Checks

1. `3학년`을 선택하고 출판사를 `와이비엠 최희경`으로 고른 뒤 `현재 단원 저장`을 누르면, 해당 단원 문서와 `teachers.gradePublishers["3"]`에 모두 같은 출판사가 반영된다.
2. `3학년`에 `와이비엠 최희경`, `4학년`에 `동아출판 윤여범`을 저장한 뒤 학년을 오가면 각 학년 드롭다운 값이 서로 섞이지 않는다.
3. 현재 학년을 `4학년`으로 두고 엑셀 업로드를 하면, 엑셀로 반영된 모든 `4학년` 단원 문서에 같은 `publisher`가 저장된다.
4. 현재 학교가 `도촌초등학교`일 때 출판사 검색 결과에는 `도촌초등학교` 세트와 다른 학교 세트가 함께 보이고, `도촌초등학교` 결과에는 `우리 학교` 표시가 붙는다.
5. 출판사 기반 복사에서 같은 단어가 이미 현재 학교 세트에 있으면 중복 저장되지 않고, 새 단어 수와 중복 건너뜀 수가 완료 메시지에 정확히 표시된다.
6. `gradePublishers["4"]`는 비어 있지만 기존 `4학년 2단원` 세트 문서에만 `publisher`가 있는 경우, 세트를 불러오면 editor에는 그 출판사가 초기값으로 보이되 자동 저장되지는 않고, 사용자가 저장을 눌러야 학년 메타데이터에 확정 반영된다.

## Out Of Scope

- 학생 화면에서 출판사 표시 또는 선택
- 출판사 자유 입력 기능
- 학교 간 권한 공유 관리
- 복사 결과의 부분 단원 선택
