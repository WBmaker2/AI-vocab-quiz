# English Vocabulary Typing Game Design

## Summary

학생 모드에 `영어 단어 타자 게임`을 추가한다. 이 게임의 목적은 빠른 타자 실력 경쟁이 아니라, 단원 핵심 단어를 `듣고`, `뜻을 보고`, `직접 영어 철자를 입력해 보는 경험`을 반복하게 하는 것이다.

이번 설계는 기존 추천안 중 `듣고 쓰기 레이스`를 기본으로 하되, 학생 난이도를 낮추기 위해 `한국어 뜻을 함께 보여주고`, `영어 발음은 TTS로 들려주는 방식`으로 확정한다. 즉, 학생은 매 문제마다 한국어 뜻을 확인하고, 영어 발음을 들은 뒤, 해당 영어 단어를 키보드로 입력한다.

## Goals

- 학생이 단원 핵심 영어 단어를 직접 타이핑해 보는 활동을 제공한다.
- 철자를 외워 쓰는 부담을 줄이기 위해 한국어 뜻과 영어 발음을 함께 제공한다.
- 빠른 속도보다 `정확한 철자 입력 경험`을 더 중요한 목표로 둔다.
- 현재 프로젝트의 다른 학생 활동과 비슷한 구조로 자연스럽게 들어가도록 설계한다.
- 학생이 지루하지 않도록 콤보, 라운드 진행, 즉시 피드백, 완료 보상을 넣는다.

## Non-Goals

- 전문 타자 연습 앱처럼 분당 타수 중심의 경쟁 시스템을 만들지 않는다.
- 문장 단위 타자나 자유 입력형 서술 문제는 이번 범위에 넣지 않는다.
- 교사용 별도 실시간 세션형 활동으로 시작하지 않는다.
- 오타 하나하나를 과도하게 감점하는 스트레스형 설계는 사용하지 않는다.

## Chosen Approach

이번 문서는 `뜻 + TTS + 단어 직접 입력` 구조를 채택한다.

문제마다 학생이 보게 되는 정보는 다음과 같다.

- 한국어 뜻
- 영어 발음 듣기 버튼
- 입력창
- 제출 버튼

학생은 한국어 뜻을 보고, TTS로 발음을 들은 뒤, 해당 영어 단어를 직접 입력한다. 입력이 끝나면 즉시 정답 여부가 보이고, 정답이면 짧은 축하음과 함께 다음 문제로 넘어간다. 단원이 모두 끝나면 긴 축하음과 결과 화면을 보여준다.

이 접근을 선택한 이유는 다음과 같다.

- 학생이 의미와 발음을 동시에 단서로 받아 단어를 떠올리기 쉽다.
- 철자 쓰기 경험이 핵심이라 교육 목적과 맞는다.
- 기존 듣기/말하기/짝 맞추기/낚시 게임의 구조와 잘 이어진다.
- 너무 어려운 받아쓰기처럼 느껴지지 않으면서도 학습 밀도는 높다.

## Alternatives Considered

### 1. 영어 발음만 듣고 완전 받아쓰기

- 장점: 가장 강한 듣기-쓰기 훈련이 된다.
- 단점: 초등학생에게는 난도가 높고 좌절감이 커질 수 있다.
- 단점: 첫 버전으로는 오답률이 너무 높아 게임 몰입이 떨어질 수 있다.

### 2. 한국어 뜻만 보고 입력

- 장점: 구현이 단순하고 직관적이다.
- 장점: 의미와 철자를 바로 연결하기 좋다.
- 단점: 발음 경험이 빠져 현재 프로젝트의 듣기 요소와 연결감이 약해진다.

### 3. 뜻 + TTS + 직접 입력

- 장점: 난이도가 적절하고 학습 단서가 충분하다.
- 장점: 학생이 `보고`, `듣고`, `쓰는` 세 가지 경험을 함께 하게 된다.
- 장점: 게임성과 교육성의 균형이 가장 좋다.

이번 문서에서는 `3. 뜻 + TTS + 직접 입력`을 채택한다.

## User Experience

### 1. 진입 화면

학생 화면의 활동 버튼 영역에 새 게임 진입 버튼을 추가한다.

버튼 예시:

- `영어 단어 타자 게임`

진입 후 시작 화면에서는 다음 정보를 보여준다.

- 선택된 단원 또는 단원 조합
- 총 문제 수
- 게임 설명
- `게임 시작`
- `홈으로`

설명 문구 예시:

`뜻을 보고 발음을 들은 뒤 영어 단어를 직접 입력해 보세요. 빠른 타자보다 정확하게 써 보는 것이 더 중요합니다.`

### 2. 플레이 화면

플레이 화면은 한 문제에 집중하기 쉬운 단일 카드 구조로 구성한다.

표시 요소:

- 현재 문제 번호와 전체 문제 수
- 진행 바
- 한국어 뜻
- `발음 듣기`
- 영어 단어 입력창
- `입력 완료`
- 피드백 메시지
- 콤보 표시

기본 흐름:

1. 문제 화면이 열린다.
2. 한국어 뜻이 보인다.
3. 시작 시 영어 발음을 자동으로 1회 읽어 준다.
4. 학생은 필요하면 `발음 듣기` 버튼으로 다시 들을 수 있다.
5. 학생이 영어 단어를 입력하고 `입력 완료`를 누른다.
6. 정답이면 짧은 축하음, 콤보 증가, 다음 문제로 이동한다.
7. 오답이면 오답 안내를 보여주고 다시 시도할 수 있게 한다.

### 3. 난이도 완화 장치

학생이 너무 어렵게 느끼지 않도록 아래 장치를 둔다.

- 한국어 뜻은 항상 보인다.
- 첫 발음은 자동 재생한다.
- `발음 듣기` 버튼으로 재청취할 수 있다.
- 오답 시 바로 게임 오버가 아니라 재도전 기회를 준다.
- 너무 많이 틀리면 `힌트` 또는 `다음 문제` 흐름을 열 수 있도록 설계 여지를 남긴다.

첫 버전 권장 규칙:

- 문제당 최대 3회 입력 시도
- 3회 실패 시 정답을 짧게 보여주고 다음 문제로 이동
- 정답 공개 후에도 학생이 지나치게 오래 멈추지 않도록 자동 진행

### 4. 정답 판정 경험

정답 판정은 학생이 위축되지 않도록 단순하고 명확하게 처리한다.

- 대소문자는 구분하지 않는다.
- 앞뒤 공백은 무시한다.
- 중간의 여러 공백은 하나로 정규화한다.

정답 시:

- `정답입니다!`
- 짧은 축하음
- 콤보 증가
- 입력창 잠금 후 짧은 전환 애니메이션

오답 시:

- `다시 한 번 써 보세요.`
- 입력창 유지
- 발음 다시 듣기 유도
- 필요 시 철자 힌트 버튼 활성화

### 5. 힌트 시스템

힌트는 학생이 끝까지 직접 써 보도록 돕는 보조 장치여야 한다.

첫 버전 추천 힌트:

- 첫 글자 공개
- 글자 수 표시

예시:

- 뜻: `네 번째`
- 힌트: `f _ _ _ _ h`

힌트는 점수에 약간의 페널티를 줄 수 있지만, 너무 크게 깎지 않는다. 목표는 학습 지속이지 경쟁 압박이 아니기 때문이다.

### 6. 완료 화면

단원을 모두 마치면 결과 화면을 보여준다.

표시 항목:

- 최종 점수
- 맞힌 개수
- 힌트 사용 횟수
- 평균 입력 시간
- 최고 콤보
- 축하 메시지

완료 시에는 기존 규칙과 맞춰 `긴 축하음`을 재생한다.

버튼 예시:

- `다시 하기`
- `다른 단원 선택`
- `홈으로`

## Game Rules

### Round structure

- 한 문제당 단어 1개를 입력한다.
- 기본 문제 수는 선택된 단어 수 전체를 사용한다.
- 중복 없이 한 번씩 출제한다.

### Start behavior

- 문제 시작 시 영어 발음을 자동으로 1회 읽는다.
- 학생이 원할 때만 `발음 듣기`로 다시 들을 수 있다.

### Answer attempts

- 기본 3회까지 입력 시도 가능
- 3회 실패 시 정답 공개 후 다음 문제로 이동

### Combo

- 연속 정답이면 콤보 증가
- 오답 3회 누적 또는 힌트 사용 시 콤보는 끊긴다

## Scoring

점수는 `빠른 타자 속도`보다 `정확도` 중심으로 계산한다.

권장 점수 구조:

- 기본 정답 점수: `100점`
- 빠르게 맞히면 시간 보너스: `최대 20점`
- 콤보 보너스: `콤보당 +5점`, 상한 있음
- 힌트 사용 시 소폭 감점: `-10점`
- 3회 실패 후 자동 공개는 점수 없음

예시:

- 바로 정답 + 힌트 없음 + 빠른 입력: `120점`
- 정답이지만 힌트 사용: `90점`
- 3회 실패: `0점`

이 구조는 정확한 입력을 우선 보상하면서도 지나친 속도 경쟁으로 흐르지 않게 해 준다.

## Content Selection

출제 단어는 현재 학생이 불러온 단원 기준으로 구성한다.

확장 가능성:

- 단일 단원 타자 게임
- 여러 단원 합치기
- 틀린 단어 다시 복습 모드

첫 버전은 현재 다른 학생 게임들과 일관성을 위해 `선택된 단원 또는 현재 불러온 단어 세트` 기준으로 진행하는 것을 권장한다.

## Audio Behavior

- 문제 시작 시 TTS 자동 1회 재생
- `발음 듣기` 버튼 클릭 시에만 추가 재생
- 정답 시 짧은 축하음
- 게임 완료 시 긴 축하음

오답일 때는 불필요한 경고음을 크게 쓰지 않는다. 학생이 틀렸다는 부담보다 다시 시도할 여지를 주는 것이 더 중요하다.

## Teacher Considerations

교사 화면에서는 별도 실시간 진행 UI 없이 기존 단어 세트 관리 구조를 그대로 사용한다.

교사 입장에서 중요한 점:

- 등록한 단어로 바로 타자 게임이 가능해야 한다.
- 학생이 반복해서 단어를 써볼 수 있어야 한다.
- 추후 리더보드나 개인 성장 기록으로 확장하기 쉬워야 한다.

첫 버전에서는 교사용 별도 설정 없이 바로 사용할 수 있도록 단순하게 유지한다.

## Data And Persistence

첫 버전은 두 가지 확장 방향을 고려해 설계한다.

### 1. 결과만 표시하는 단순 버전

- 점수, 정답 수, 힌트 사용 수를 세션 메모리에서 계산
- 완료 화면에만 보여주고 저장하지 않음

### 2. 이후 확장 가능한 저장형 버전

- 학생 이름 기준 최고 점수 저장
- 주간/월간/연간 리더보드 연결
- 학생 개인 성장 기록에 타자 게임 활동 추가

현재 프로젝트 흐름상, 추후에는 `student progression`과 `activity leaderboard` 체계로 자연스럽게 연결할 수 있게 필드 구조를 맞추는 것이 좋다.

권장 결과 필드:

```json
{
  "score": 840,
  "correctCount": 8,
  "hintUsedCount": 2,
  "failedCount": 2,
  "averageInputSeconds": 4,
  "bestCombo": 4
}
```

## Accessibility And Usability

- 입력창은 화면 진입 시 자동 포커스를 받는 것이 좋다.
- Enter 키로 제출할 수 있어야 한다.
- 모바일이나 태블릿에서는 가상 키보드가 자연스럽게 올라오도록 한다.
- TTS 버튼과 제출 버튼은 시각적으로 명확히 구분한다.
- 너무 화려한 애니메이션보다 문제 집중이 잘 되는 구조를 우선한다.

## Risks

### 1. 너무 타자 시험처럼 느껴질 위험

속도 점수 비중이 커지면 학습보다 압박이 커질 수 있다.

대응:

- 정확도 비중을 높인다.
- 힌트와 재도전 기회를 둔다.

### 2. 너무 쉬워서 금방 지루해질 위험

뜻과 발음을 동시에 주면 난도가 지나치게 낮을 수 있다.

대응:

- 콤보
- 시간 보너스
- 힌트 최소화
- 상위 난이도 모드 여지 확보

### 3. 입력 실수 스트레스

초등학생은 철자 하나만 틀려도 크게 좌절할 수 있다.

대응:

- 오답 후 즉시 탈락시키지 않는다.
- `다시 듣기`와 `힌트` 흐름을 제공한다.

## Implementation Notes

- 기존 학생 활동 선택 구조에 새 버튼을 추가한다.
- 기존 speech/TTS 유틸을 재사용한다.
- 기존 축하음 유틸을 재사용한다.
- 결과 화면 패턴은 `짝 맞추기`, `단어 낚시`와 비슷하게 맞춘다.
- 이후 리더보드 확장을 고려해 결과 객체 구조는 공통화 가능하게 설계한다.

## Acceptance Checks

- 학생이 단어 세트를 불러온 뒤 타자 게임을 시작할 수 있다.
- 각 문제에서 한국어 뜻이 보인다.
- 각 문제 시작 시 영어 발음이 자동으로 1회 재생된다.
- 학생은 `발음 듣기`로 다시 들을 수 있다.
- 학생은 영어 단어를 입력하고 제출할 수 있다.
- 정답이면 즉시 축하 피드백과 함께 다음 문제로 넘어간다.
- 오답이어도 재도전 기회가 있다.
- 모든 문제 완료 후 점수와 결과 요약 화면이 나온다.
- 게임 완료 시 긴 축하음이 재생된다.

## Open Follow-Ups

- 리더보드를 첫 버전부터 붙일지, 후속 버전으로 둘지 결정
- 학생 개인 성장 기록에 `typing` 활동을 추가할지 결정
- 힌트 종류를 `첫 글자 공개`까지만 둘지, `글자 수 표시`까지 넣을지 결정
