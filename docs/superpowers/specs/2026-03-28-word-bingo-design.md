# 단어 빙고 게임 설계 문서

작성일: 2026-03-28

## 목표

기존 `듣기 퀴즈`, `말하기 퀴즈`, `짝 맞추기 게임`과 다른 감각의 교실 전체 참여형 게임으로 `단어 빙고`를 추가한다.

핵심 목표는 다음과 같다.

- 학급이 여러 기기에서 동시에 참여할 수 있어야 한다.
- 기존 단어 세트(Firebase `vocabularySets`)를 그대로 재활용해야 한다.
- 교사가 수업을 직접 진행하는 감각을 살려야 한다.
- 교사가 단어를 읽는 방식은 두 가지를 모두 지원해야 한다.
  - 직접 읽기
  - 버튼 클릭 후 TTS로 읽기
- 교사가 선택한 단어만 학생이 체크할 수 있어야 한다.
- 학생이 실수로 다른 단어를 눌러도 체크되지 않도록 해야 한다.
- 학생 빙고판에는 영어 단어가 보여야 한다.

## 추천 접근

추천 구조는 `교사 호스트 1명 + 학생 다수 참여형 실시간 빙고 세션`이다.

이 구조를 선택한 이유:

- 현재 앱은 Firebase를 이미 사용하고 있어, 세션 동기화에 Firestore를 자연스럽게 활용할 수 있다.
- 학생별 랜덤 빙고판, 교사의 호출 단어, 현재 차례, 승자 순서 같은 정보를 실시간으로 묶기 쉽다.
- 교실 수업에서 가장 중요한 “교사 통제력”과 “학생 동시 참여”를 동시에 만족한다.

## 고려한 대안

### 1. 교사 화면 1대 + 학생은 종이 빙고판

가장 단순하지만, 앱 안에서 학생 참여 데이터를 모을 수 없고 경쟁형 게임으로 확장하기 어렵다.

### 2. 학생 각자 단독 빙고 게임

구현은 쉽지만, `학급이 함께 진행`한다는 요구와 맞지 않는다.

### 3. 교사 호스트형 실시간 세션

교사가 문제를 진행하고 학생은 같은 세션에 들어와 동시에 반응한다. 이번 요구와 가장 잘 맞는 구조라서 이 방향을 채택한다.

## MVP 범위

첫 버전은 아래 범위로 제한한다.

- 게임 종류: `단어 빙고`
- 진행 방식: `교사 호스트형 실시간 세션`
- 단어 소스: 교사가 현재 불러온 단어 세트 1개
- 참여자: 같은 수업에 참여하는 학생 여러 명
- 빙고판: 학생마다 랜덤 생성
- 승리 조건: `3빙고`
- 결과 표시: 교사/학생 모두 실시간으로 확인 가능
- 교사 호출 방식:
  - `직접 읽기`: 교사가 단어를 클릭해 현재 호출 단어를 확정하고 직접 읽음
  - `TTS 읽기`: 교사가 단어를 클릭하면 호출 확정과 함께 TTS 재생
  - `랜덤 단어 뽑기`: 애니메이션 후 단어 확정, TTS 재생

이번 MVP에서 제외:

- 여러 단원 동시 빙고
- 반 전체 누적 시즌전
- 서버 함수 기반 부정행위 방지
- 출석/학급 roster 관리
- 음성 인식으로 학생 응답 검증

향후 확장 후보:

- `학생 랜덤 발표 모드`
  - 교사가 랜덤 학생을 뽑는다
  - 뽑힌 학생이 자기 빙고판의 영어 단어를 하나 선택해 읽는다
  - 다른 학생 보드에 같은 영어 단어가 있으면 함께 체크된다

## 핵심 사용자 흐름

## 1. 교사 시작 흐름

1. 교사는 교사 모드에서 기존처럼 학교, 학년, 단원 단어 세트를 불러온다.
2. `단어 빙고 수업 시작` 버튼을 누른다.
3. 빙고 세션 설정에서 아래를 정한다.
   - 읽기 방식: `직접 읽기` 또는 `TTS 읽기`
   - 판 크기: 기본 `4x4` 추천
4. 앱은 짧은 참여 코드와 현재 세션 상태를 만든다.
5. 교사는 학생들에게 참여 코드 또는 참여 화면을 안내한다.

## 2. 학생 참여 흐름

1. 학생은 학생 모드에서 새 `학급 빙고 게임` 버튼을 누른다.
2. 학생은 다음 정보를 입력한다.
   - 학생 이름
   - 참여 코드
3. 세션 참여가 성공하면, 학생별 랜덤 빙고판이 생성된다.
4. 학생은 교사가 호출한 단어를 기다린다.

## 3. 게임 진행 흐름

1. 교사는 호스트 화면의 단어 목록에서 하나를 클릭한다.
2. 시스템은 그 단어를 현재 호출 단어로 세션에 기록한다.
3. `직접 읽기` 모드면 교사가 직접 읽는다.
4. `TTS 읽기` 모드면 앱이 해당 영어 단어를 읽어 준다.
5. 학생 화면에서는 현재 호출 단어와 일치하는 칸만 활성화된다.
6. 학생은 활성화된 칸만 눌러 체크할 수 있다.
7. 다른 칸은 비활성화되어 실수로 눌러도 반응하지 않는다.
8. 3빙고가 되면 해당 학생은 `빙고!` 상태가 되고, 교사 화면에 학생별 1빙고, 2빙고, 3+빙고 현황이 반영된다.
9. 3빙고 이후에도 교사가 수업을 종료하기 전까지 학생은 계속 체크할 수 있다.

## 4. 종료 흐름

첫 번째 학생이 3빙고를 달성해도 교사는 다음 중 하나를 선택할 수 있다.

- 수업 종료
- 계속 진행해서 4빙고, 5빙고 이상을 확인

MVP에서는 기본적으로 `3빙고 달성 시 현황 표시 + 교사 수동 종료` 흐름을 사용한다.

## 교사 통제 규칙

이 기능에서 가장 중요한 규칙은 `교사가 단어를 선택해야만 학생이 체크할 수 있다`는 점이다.

세부 규칙:

- 학생은 자기 빙고판의 아무 칸이나 자유롭게 체크할 수 없다.
- 현재 세션의 `activeWordId` 또는 `activeCallId`와 일치하는 칸만 클릭 가능하다.
- 이미 체크된 칸은 다시 클릭할 수 없다.
- 같은 단어를 교사가 다시 호출하더라도 중복 체크는 일어나지 않는다.
- 교사가 단어를 클릭하거나 랜덤/TTS로 호출해도 칸은 자동 체크되지 않는다.
- 교사가 단어를 클릭하지 않은 상태에서는 학생 보드의 모든 칸이 비활성화된다.

이 규칙 덕분에 교사가 직접 읽는 경우에도 학생이 실수로 다른 단어를 고르지 못한다.

## 빙고판 설계

기본 추천은 `4x4` 빙고판이다.

이유:

- 초등학생 수업 시간에 진행하기 적당한 길이
- 3x3보다 게임성이 좋고 5x5보다 부담이 적음
- 일반적인 단원 단어 수와도 잘 맞음

판 생성 규칙:

- 학생별로 동일 단어 풀에서 랜덤 추출
- 중복 없이 배치
- 한 세션 안에서는 학생마다 다른 배치 가능
- 단어 수가 부족하면 판 크기를 자동으로 줄인다

권장 축소 규칙:

- 단어 수 16개 이상: `4x4`
- 단어 수 9개 이상: `3x3`
- 9개 미만: 세션 시작 불가, 교사에게 안내

## 표시 형태

첫 버전은 `영어 보기 빙고판`을 기본으로 둔다.

즉:

- 교사는 영어 단어를 읽는다
- 학생 빙고판에도 영어 단어가 보인다

이 방식의 장점:

- 교사가 단어를 클릭해 직접 읽는 흐름과 가장 자연스럽게 맞는다
- 이후 `학생 랜덤 발표 모드`를 붙이기 쉽다
- 학생이 자기 보드에서 직접 영어 단어를 찾아 읽는 확장이 가능하다

향후 확장 가능 옵션:

- 뜻 보기 빙고판
- 영어/뜻 혼합 빙고판

하지만 MVP에서는 포함하지 않는다.

## Firebase 데이터 구조

새 컬렉션을 추가한다.

### 1. `bingoSessions/{sessionId}`

세션 메타데이터를 저장한다.

예상 필드:

- `sessionCode`
- `teacherUserId`
- `teacherName`
- `schoolId`
- `schoolName`
- `grade`
- `unit`
- `publisher`
- `mode` (`manual` | `tts`)
- `boardSize`
- `status` (`waiting` | `live` | `finished`)
- `activeWordId`
- `activeWordText`
- `callSequence`
- `calledWordIds`
- `vocabularyItems`
- `winnerStudentIds`
- `createdAt`
- `updatedAt`

### 2. `bingoSessions/{sessionId}/players/{playerId}`

학생별 보드와 진행 상태를 저장한다.

예상 필드:

- `studentName`
- `studentNameNormalized`
- `boardCells`
- `markedWordIds`
- `bingoLines`
- `hasBingo` (`bingoLines >= 3` 기준)
- `bingoRank`
- `joinedAt`
- `updatedAt`

## 실시간 동기화 전략

교사와 학생 모두 Firestore `onSnapshot` 기반으로 세션을 구독한다.

- 교사 화면은 세션 메타데이터 + 참가자 하위 컬렉션을 본다.
- 학생 화면은 자신의 player 문서 + 세션 메타데이터를 본다.

핵심 동기화 대상:

- 현재 호출 단어
- 호출 순서
- 학생 체크 상태
- 빙고 달성 여부
- 승자 순서

## 점수/승자 처리

MVP의 경쟁 규칙은 단순하게 둔다.

- 먼저 한 줄 빙고를 만든 학생이 우선
- 같은 시점 충돌은 `updatedAt` 또는 서버 기록 순서 기준
- 교사 화면에는 `1등, 2등, 3등...` 순서로 표시
- 학생 화면에는 본인 상태와 현재 승자 보드만 간단히 노출

리더보드 영구 저장은 첫 버전에서는 필수가 아니다.

이유:

- 수업용 즉시 경쟁으로도 충분히 재미를 줄 수 있다
- 기존 매칭 리더보드 구조와 목적이 다르다
- 먼저 게임 자체를 안정화하는 것이 우선이다

## UI 구성

## 1. 교사 호스트 화면

필수 요소:

- 세션 정보 카드
  - 학교
  - 학년
  - 단원
  - 참여 코드
  - 참여 학생 수
- 읽기 방식 표시
- 단어 호출 보드
- 현재 호출 단어 표시
- TTS 재생/직접 읽기 안내
- 승자 현황
- 수업 종료 버튼

## 2. 학생 빙고 참여 화면

필수 요소:

- 학생 이름 입력
- 참여 코드 입력
- 참여 버튼
- 오류 안내 문구

## 3. 학생 빙고판 화면

필수 요소:

- 현재 호출된 영어 단어 표시
- 내 빙고판
- 현재 상태 표시
  - 대기 중
  - 체크 가능
  - 빙고 달성
- 간단한 승자 현황

## 오류 및 예외 처리

- 참여 코드가 잘못되면 입장 불가
- 세션이 종료된 후에는 참여 불가
- 단어 수가 판 크기보다 적으면 교사 시작 차단
- 같은 학생 이름 중복 입장은 첫 버전에서는 허용하지 않는 방향을 추천
- 네트워크 지연으로 학생이 늦게 받아도, 현재 `activeWordId` 기준만 체크 가능

## 보안 및 무결성

현재 앱 구조상 Firestore 직접 쓰기 기반이므로 완전한 부정행위 방지는 어렵다.

첫 버전 방침:

- 학생은 자기 player 문서만 수정 가능
- 교사만 세션 메타데이터 수정 가능
- 학생은 현재 호출 단어와 일치하는 체크만 허용하는 방향으로 rules 설계

다만 완전한 검증이 필요하면 이후 단계에서 Cloud Functions로 옮겨야 한다.

## 검증 기준

- 교사가 세션을 시작할 수 있다
- 학생이 코드로 세션에 참여할 수 있다
- 학생마다 랜덤 빙고판이 생성된다
- 교사가 단어를 클릭하면 현재 호출 단어가 모든 학생에게 반영된다
- 직접 읽기 모드에서는 교사 클릭만 하고 음성은 자동 재생되지 않는다
- TTS 모드에서는 교사 클릭 시 음성이 재생된다
- 학생은 교사가 선택한 단어에 해당하는 칸만 체크할 수 있다
- 한 줄 완성 시 교사와 학생 화면 모두에서 빙고 상태가 반영된다
- 교사 화면에서 승자 순서가 누적된다

## 최종 결론

이 기능은 `실시간 교실 빙고 세션`으로 설계하는 것이 가장 적절하다.

핵심은 다음 두 가지다.

- 교사가 수업을 통제하는 호스트 역할을 가진다
- 학생은 교사가 호출한 단어에만 반응할 수 있는 안전한 빙고판을 가진다

이 설계는 현재 앱의 Firebase 기반 구조와 잘 맞고, 기존 단어 세트도 그대로 재활용할 수 있다.

또한 빙고판이 영어 단어 기준이므로, 이후 단계에서 `학생 랜덤 발표 모드`를 추가할 때 별도 보드 구조 변경 없이 확장할 수 있다.
