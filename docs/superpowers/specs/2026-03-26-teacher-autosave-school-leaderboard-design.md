# Teacher Autosave And School Leaderboard Design

## Summary

교사 모드의 저장 흐름을 더 자연스럽게 만들고, 학생 모드의 막힘을 줄이며, 짝 맞추기 리더보드에 학교 전체 통합 보드를 추가한다. 이번 변경은 기존 수업 흐름을 유지하면서 UX를 보강하는 것이 목표다.

## Goals

- 교사 모드에서 단어 추가, 수정, 삭제 시 자동 저장이 되게 한다.
- 교사 모드의 단원 선택을 저장된 단원 중심의 드롭다운으로 바꾼다.
- 말하기 퀴즈에서 3회 연속 실패 또는 음성 인식 실패가 반복되면 다음 문제로 넘어갈 수 있게 한다.
- 짝 맞추기 완료 화면의 리더보드에 `우리학교 단어 왕` 탭을 추가한다.
- 학생 화면에서는 교사용 편집 버튼을 보이지 않게 한다.

## Non-Goals

- 교사 모드의 전체 저장 구조를 새로 설계하지 않는다.
- 듣기/말하기 리더보드를 새로 만들지 않는다.
- 학교 전체 리더보드를 기간별 보드로 확장하지 않는다.

## Design Decisions

### 1. Teacher Autosave

- 자동 저장 대상은 현재 선택된 `학년 + 단원 + 출판사 + 공개 상태 + 단어 목록`이다.
- 자동 저장은 단어 추가, 단어 수정, 단어 삭제 후 debounce 방식으로 실행한다.
- 저장 가능 조건:
  - Firebase 연결 가능
  - 교사 로그인 완료
  - 학년 선택 완료
  - 단원 선택 완료
  - 단어 목록 1개 이상 또는 현재 단원에 대한 명시적 삭제 상태
- 수동 `현재 단원 저장` 버튼은 유지한다. 자동 저장 실패 시 교사가 수동으로 다시 저장할 수 있어야 한다.
- 자동 저장 상태는 `자동 저장 중...`, `자동 저장됨`, `자동 저장 실패`처럼 짧은 상태 메시지로 표시한다.

### 2. Teacher Unit Picker

- 교사 모드의 단원 필드는 기본적으로 드롭다운을 사용한다.
- 드롭다운에는 현재 선택 학년에 대해 이미 등록된 단원들이 표시된다.
- 드롭다운의 마지막 옵션으로 `+ 새 단원 만들기`를 둔다.
- `+ 새 단원 만들기` 선택 시에만 새 단원 입력 필드를 보여준다.
- 이렇게 하면 기존 단원은 쉽게 고르고, 새 단원도 계속 추가할 수 있다.

### 3. Speaking Quiz Escape Hatch

- 말하기 퀴즈는 문제별 `failedAttempts`를 관리한다.
- 다음 경우 `다음 문제` 버튼을 활성화한다.
  - 정답 처리
  - 오답 3회 이상
  - `no-speech` 또는 유사 음성 인식 실패가 3회 이상
  - STT 설정 오류로 더 진행하기 어려운 상태
- 정답을 맞히거나 다음 문제로 넘어가면 해당 카운트는 초기화한다.

### 4. School-Wide Matching Leaderboard

- 기존 탭:
  - `이 주의 영단어 왕`
  - `이 달의 영단어 왕`
  - `올해의 영단어 왕`
- 새 탭:
  - `우리학교 단어 왕`
- `우리학교 단어 왕`은 학년 구분 없이 같은 학교 전체 학생 중 점수 상위 20명을 보여준다.
- 정렬 규칙:
  - 점수 내림차순
  - 동점이면 더 짧은 시간 우선
  - 그래도 같으면 먼저 저장된 기록 우선
- 구현은 매칭 점수 저장 시 기존 주간/월간/연간 저장과 함께 별도 `school_all` scope도 upsert 하는 방식으로 간다.

### 5. Student-Only Screen Cleanup

- 학생 화면에서는 `단어 카드 편집`, `단어 세트 수정`, 교사 화면 진입용 버튼을 숨긴다.
- 학생 결과 화면에는 학습 관련 액션만 남긴다.

## Data Model Impact

### Matching Leaderboard

- 기존 periodType: `week`, `month`, `year`
- 추가 periodType: `school_all`
- `school_all`은 periodKey를 고정값 `all-time`으로 사용한다.
- scopeKey 예시:
  - `schoolId__3__week__2026-W13`
  - `schoolId__3__month__2026-03`
  - `schoolId__3__year__2026`
  - `schoolId__all__school_all__all-time`

`school_all`은 학년 구분이 없으므로 grade를 저장할 때는 문서에는 실제 학생 학년을 유지하되, scope 계산은 `all`을 사용한다. 화면에는 학생 이름 옆에 학년을 같이 보여준다.

## UI Impact

### Teacher Mode

- `학년과 단원 선택` 카드에서 단원 입력 필드를 드롭다운 + 새 단원 입력 보조 필드로 바꾼다.
- 자동 저장 상태 문구를 단원 카드 안에 노출한다.

### Speaking Quiz

- `다음 문제` 버튼 활성화 조건을 명확히 하고, 비활성 설명 문구를 줄인다.

### Matching Result

- 탭형 리더보드에 `우리학교 단어 왕` 탭 추가
- 학교 전체 보드에서는 학생 이름, 학년, 점수, 시간 표시

## Error Handling

- 자동 저장 실패 시 현재 편집 내용은 유지하고 에러만 표시한다.
- `school_all` 리더보드 저장 일부 실패 시 기존 period save failure와 같은 패턴으로 안내한다.
- 음성 인식 실패 카운트는 `지원하지 않음`과 `권한 거부` 같은 설정 오류와 구분한다.

## Verification

- 교사 모드에서 단어 추가 후 자동 저장되고 새로고침 뒤 유지되는지 확인
- 기존 단원 선택과 새 단원 생성이 모두 가능한지 확인
- 말하기 퀴즈에서 오답/미인식 3회 후 다음 문제 버튼이 활성화되는지 확인
- 짝 맞추기 완료 후 `우리학교 단어 왕` 탭이 보이고 상위 20명 정렬이 맞는지 확인
- 학생 화면에 교사 편집 버튼이 남아 있지 않은지 확인
- `npm run build` 통과
